RGBW_Strip constructor.

Notice that several strips can be connected, but they must be connected on different RMT channels. You shouldn't need to call this class directly, but you can use it to drive other led strips not currently supported, with custom duration values.

Parameters
[in]gpioNum,LedStrip GPIO.
[in]pixelCount,Numberof leds.
[in]channel,RMTchannel to use. See https://docs.espressif.com/projects/esp-idf/en/latest/api-reference/peripherals/rmt.html#enumerations
[in]t0h,highvalue duration for bit 0, expressed in RMT ticks.
[in]t0l,lowvalue duration for bit 0, expressed in RMT ticks.
[in]t1h,highvalue duration for bit 1, expressed in RMT ticks.
[in]t1l,lowvalue duration for bit 0, expressed in RMT ticks.

void RGBW_Strip::clear ( )
virtual

Clear all the pixel colors for an RGBW strip.

This sets all the pixels to off which is no brightness for all of the color channels. The LEDs are not actually updated until a call to show().
